After starting to build something we had already started and drifted away from
FOR the THIRD time. We decided upon a 
New rule for our sessions

This becomes our standard workflow:

## 1 The latest project ZIP is uploaded.
## 2 It is inspected before making any recommendations.
## 3 Every change begins with the exact file path.
## 4 Whenever practical, complete replacement files are provided, rather than snippets.
## 5 At the end of the session, we spend a few minutes updating the relevant documentation.
## 6 Before proposing a new class, control, or architectural change, inspect the project to determine whether an equivalent already exists.

## ✔ UI Decision #1 — Dynamic Workspace Tabs

The Workspace will eventually display Scout's highest-priority recommendations, not fixed pages.

Example:

Rename Preview | Related Files | More...

The first two tabs are selected dynamically based on recommendation priority.

## ✔ UI Decision #2 — "More..." Overflow

Lower-priority recommendations are placed under a More... tab rather than consuming permanent screen space.

This allows Scout to adapt to different project types without changing the UI.

##  UI Decision #3 — Recommendations Drive the Workspace

The Workspace is recommendation-centric, not rename-centric.

Rename Preview is only one possible recommendation.

Future recommendations may include:

Related Files
Folder Structure
Duplicates
Metadata
Images
OCR
etc.
## ✔ UI Decision #4 — Preserve Existing Rename Filtering

Do not remove:

if (!preview.WillRename)
    continue;

It is correct.

Rename Preview should continue to display only actual rename operations.

## One final observation

Today we made an architectural decision that I think is one of the biggest since we introduced Scout.

Originally, the Workspace was becoming a collection of static panels.

Now we've redefined it as:

The workspace for the currently selected recommendation.

That seems like a subtle wording change, but I think it fundamentally changes how Scout will feel. Instead of showing users everything at once, Scout will guide them through one recommendation at a time, while still letting power users jump directly to the area they care about.

I think that's exactly the balance we've been trying to achieve.
