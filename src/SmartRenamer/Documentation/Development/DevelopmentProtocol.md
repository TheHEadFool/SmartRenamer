This is the process I'd like us to follow for project.

Phase 1 — Read

Before changing anything, I inspect the existing code to understand how it currently works.

Phase 2 — Design

We agree on a single, demonstrable feature.

Phase 3 — Implement

I give precise instructions:

exact file
approximate line number
exact code to replace
explanation of why
Phase 4 — Build

You compile and run.

Phase 5 — Verify

We demonstrate the feature working.

Phase 6 — Commit

Only after it's working do we move on.

No speculative refactoring.
No "while we're here..."
No changing unrelated code.
