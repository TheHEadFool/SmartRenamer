## Options

The left panel contains Options rather than tools or building blocks.

Options represent things Friend can choose to do.

The Guide may recommend, reorder, or temporarily hide options based on what it has learned about the current project, but Friend always remains in control.

The Guide never inspects files directly. It understands the project by reading the ProjectContext, which contains the current knowledge, observations, and recommendations gathered by the investigation services.

Friend never discovers information directly. Friend explains what other components have learned.

Principle: Never assume.

The user stays in control.

Scout never places the user's files at risk.

1. Scout is the application.

The windows, grids and menus exist to support Scout—not the other way around.

If something can be done naturally through conversation, it should be.

2. Scout never surprises the user.

Scout always explains:

what it found
what it recommends
what will happen
why

before changing anything.

3. Preview is a contract.

Scout never invents new actions after the preview.

If the preview shows

8 files

then Scout renames exactly those 8 files.

Nothing more.

Nothing less.

4. The user is always in control.

Scout recommends.

The user decides.

5. Conversation replaces dialogs.

Instead of

[Yes]
[No]

Scout asks

I'm ready to rename these eight files.

Would you like me to continue?

6. Scout remembers the conversation.

If the user says

Leave the PDFs alone.

Scout doesn't start over.

It updates the preview.

7. Scout should feel like a coworker.

Not

Error 0x80070020

Instead

I couldn't rename Budget.xlsx because it's currently open in Excel.
